# Activity

Guía rápida para crear un **README de perfil profesional en GitHub**.

## ¿Dónde se coloca el README de perfil?
Para que aparezca en tu perfil cuando alguien te busque:

1. Crea un repositorio **público** con el **mismo nombre de tu usuario de GitHub**.
   - Ejemplo: si tu usuario es `juanperez`, el repo debe llamarse `juanperez`.
2. Marca la opción **"Add a README file"**.
3. Edita ese `README.md` y coloca tu presentación profesional.
4. Guarda cambios y revisa tu perfil: ese contenido saldrá arriba de todo.

## Estructura recomendada (profesional)
- Encabezado con quién eres y qué haces.
- Tecnologías clave (stack).
- Proyectos destacados con links.
- Logros o certificaciones.
- Contacto (LinkedIn, portafolio, email).

## Plantilla lista para usar
Copia esto en el `README.md` de tu repo de perfil y personalízalo:

```md
# ¡Hola! Soy Tu Nombre 👋

💻 Desarrollador(a) [Frontend/Backend/Full Stack] con enfoque en [tu especialidad].

## 🚀 Sobre mí
- Actualmente trabajando en: [proyecto o rol actual]
- Aprendiendo: [tecnología]
- Intereses: [áreas]

## 🛠️ Stack
![HTML5](https://img.shields.io/badge/HTML5-E34F26?style=for-the-badge&logo=html5&logoColor=white)
![CSS3](https://img.shields.io/badge/CSS3-1572B6?style=for-the-badge&logo=css3&logoColor=white)
![JavaScript](https://img.shields.io/badge/JavaScript-F7DF1E?style=for-the-badge&logo=javascript&logoColor=black)
![React](https://img.shields.io/badge/React-20232A?style=for-the-badge&logo=react&logoColor=61DAFB)

## 📌 Proyectos destacados
- [Proyecto 1](https://github.com/tuusuario/proyecto1): breve descripción de impacto.
- [Proyecto 2](https://github.com/tuusuario/proyecto2): breve descripción de impacto.

## 📫 Contacto
- LinkedIn: [tu-linkedin]
- Portafolio: [tu-web]
- Email: [tu-email]
```

## Tips para que se vea mejor
- Usa texto corto y claro (reclutadores escanean rápido).
- Muestra impacto ("reduje tiempo", "aumenté rendimiento", etc.).
- Mantén links activos y ortografía cuidada.
- Actualízalo cada 1–2 meses.
